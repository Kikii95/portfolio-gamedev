# Asset Pipeline Engine

Système de gestion d'assets robuste pour moteurs de jeu : chargement, cache, hot-reload et résolution de dépendances.

## Contexte

Projet école GTech3, 5 semaines. Équipe de 3, rôle : architecte du pipeline core.

Les jeux modernes nécessitent une gestion d'assets sophistiquée. Ce projet implémente un pipeline production-ready pour les ressources de jeu.

## Cahier des Charges

Consignes école :
- Implémenter un système de chargement de ressources
- Gestion mémoire automatique
- Support hot-reload
- C++ moderne, documentation technique, tests unitaires

## Stack

- C++
- CMake
- Sérialisation binaire custom
- APIs système (file watching)

## Notre Implémentation

Au-delà des consignes, nous avons choisi d'implémenter :

- **Chargement async avec callbacks** : prioritisation des assets visibles à l'écran
- **Reference counting** : gestion automatique du cycle de vie, évite les memory leaks
- **Hot reload intelligent** : file watcher natif + graphe de dépendances pour recharger automatiquement les assets liés
- **Compression LZ4** : réduction de 60% de la taille des bundles packés

## Points Techniques

Resource Manager centralisé gérant :
- Enregistrement et lookup d'assets
- Budget mémoire
- Prioritisation du chargement
- Garbage collection

Pipeline de build offline pour :
- Compression textures (formats BCn)
- Optimisation mesh
- Compilation shaders
- Bundling d'assets
