  ## Architecture

  Resource Manager avec graphe de dépendances :

  ```
  ┌─────────────────────────────────────────┐
  │           Resource Manager              │
  ├──────────┬──────────┬──────────┬────────┤
  │ Registry │ Loader   │ Cache    │ Watch  │
  │ (lookup) │ (async)  │ (memory) │ (hot)  │
  └──────────┴──────────┴──────────┴────────┘
              │
              ▼
  ┌─────────────────────────────────────────┐
  │         Dependency Graph                │
  │  Texture A ──► Shader B ──► Material C  │
  └─────────────────────────────────────────┘
  ```

# Asset Pipeline Engine

Système de gestion d'assets robuste pour moteurs de jeu : chargement, cache, hot-reload et résolution de dépendances.

## Contexte

Projet école GTech3, 5 semaines. Équipe de 3, rôle : architecte du pipeline core.

Les jeux modernes nécessitent une gestion d'assets sophistiquée. Ce projet implémente un pipeline production-ready.

## Cahier des Charges École

- Implémenter un système de chargement de ressources
- Gestion mémoire automatique
- Support hot-reload
- C++ moderne, documentation, tests unitaires

## Notre Implémentation

Au-delà des consignes :

- **Chargement async avec callbacks** : prioritisation assets visibles
- **Reference counting** : évite les memory leaks
- **Hot reload intelligent** : file watcher + graphe de dépendances
- **Compression LZ4** : réduction 60% taille bundles
