# Process Tracker

Application desktop pour gérer les services de développement local. Fini les processus dupliqués, conflits de ports et incertitudes sur ce qui tourne.

## Contexte

Projet perso démarré en décembre 2025. Développé en solo sur une semaine.

Problème résolu : quand on développe avec plusieurs services (backend, frontend, database), on se retrouve souvent avec des processus dupliqués, des conflits de ports, et aucune visibilité sur ce qui tourne.

## Stack

- Tauri 2.0 (desktop framework)
- Rust (backend)
- React, TypeScript, TailwindCSS (frontend)
- Vite (build)
- Cross-platform : Windows, Linux

## Configuration

- Fichier `.ptrc` JSON pour définir les services (à commit dans Git)
- `.pt-registry.json` pour l'état runtime (à ignorer dans Git)
- Support des variables d'environnement
- Timeouts de health check personnalisables

## Fonctionnalités

Gestion services :
- Start/Stop/Restart en un clic
- Dashboard de statut global
- Health checks HTTP
- Gestion des dépendances entre services
- Historique des exécutions

Avancé :
- Détection de processus externes (services démarrés ailleurs)
- Groupes de services pour opérations batch
- Support WSL depuis Windows
- Redémarrage automatique des services crashés

## Points Techniques

Les services sont définis dans un fichier YAML/JSON simple. Le tracker surveille l'état des processus, gère l'arrêt propre et peut redémarrer automatiquement les services crashés.

Intégration WSL : peut exécuter des commandes dans les distributions WSL, permettant de gérer des services Linux depuis une GUI Windows.
