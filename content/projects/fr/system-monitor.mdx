  ## Architecture

  Design en couches séparant collecte, historique et rendu :

  ```
  /proc/* + sysinfo + nvml
           │
           ▼
  ┌─────────────────┐
  │   Collectors    │  ← CPU, Memory, GPU, Network, Disk, Process
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │ MetricsHistory  │  ← RingBuffer<60> pour sparklines
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │    Widgets      │  ← ratatui::Widget pour chaque métrique
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │ ratatui::Frame  │  ← Diff rendering, terminal output
  └─────────────────┘
  ```

  ## Métriques Performance

  - **CPU** : 0.0% utilisation en idle
  - **RAM** : ~4MB footprint
  - **Binaire** : 1.7MB (stripped, LTO enabled)
  - **Startup** : <100ms

  ## Optimisations

  - **Lazy Updates** : refresh uniquement si `tick_elapsed`
  - **Diff Rendering** : ratatui compare buffers, redessine que les cellules modifiées
  - **Batch /proc Reads** : single `refresh_all()` pour toutes métriques
  - **SmartString** : stack-allocated pour petites strings (noms process)

# System Monitor

Moniteur de performances système en terminal. Inspiré de htop/btop/nvtop mais beaucoup plus léger.

## Contexte

Projet perso démarré en novembre 2025. Développé en solo sur une semaine.

Objectif : un moniteur système minimaliste, pilotable au clavier, qui consomme presque rien tout en restant visuellement agréable.

## Ce que ça fait

**Performance** :
- Consommation quasi-nulle : 0.0% CPU, ~4MB RAM
- Binaire unique de 1.7MB
- Démarrage instantané (<100ms)

**Monitoring** :
- CPU, RAM, Swap en temps réel
- GPU NVIDIA : usage, VRAM, température, power draw
- Sparklines 60 secondes pour toutes les métriques
- Top 5 processus par consommation

**Interface** :
- 9 thèmes visuels (Matrix, Dracula, Nord, Tokyo Night, Gruvbox, etc.)
- Navigation clavier complète
- Config persistante en un keypress
