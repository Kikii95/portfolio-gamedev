# System Monitor

Moniteur de performances système en terminal. Inspiré de htop/btop/nvtop mais plus léger, écrit en Rust.

## Contexte

Projet perso démarré en novembre 2025. Développé en solo sur une semaine.

Objectif : un moniteur système minimaliste, pilotable au clavier, qui consomme presque rien tout en étant visuellement agréable.

## Stack

- Rust 1.75+
- Ratatui (TUI framework)
- sysinfo (métriques système)
- nvml-wrapper (GPU NVIDIA)
- Cross-platform : Linux, Windows, macOS, WSL2

## Fonctionnalités

Performance :
- Consommation quasi-nulle : 0.0% CPU, ~4MB RAM
- Binaire unique de 1.7MB
- Démarrage instantané (<100ms)

Monitoring :
- CPU, RAM, Swap en temps réel
- GPU NVIDIA : usage, VRAM, température, consommation
- Sparklines 60 secondes pour toutes les métriques
- Processus triés par consommation

Interface :
- 9 thèmes visuels : Matrix, Dracula, Nord, Tokyo Night, Gruvbox, Cyberpunk, Ocean, Forest, Sunset
- Navigation clavier complète
- Config persistante sauvegardée en un keypress

## Points Techniques

Design zero-overhead : les métriques sont collectées via sysinfo (APIs système natives). Le TUI ne redessine que quand les données changent.

Intégration GPU via nvml-wrapper (bibliothèque NVML de NVIDIA), accès temps réel à l'utilisation, mémoire, température et consommation.

Config stockée dans `~/.config/system-monitor/config.toml` (Linux/WSL) ou `%APPDATA%` (Windows).
