# FBX Parser

Parser custom pour le format binaire FBX, permettant l'import de modèles 3D, animations squelettiques et hiérarchies de scène.

## Contexte

Projet école GTech3, 5 semaines. Équipe de 3, rôle : implémentation du parser binaire.

Comprendre les formats de fichiers 3D est essentiel en dev jeux. Ce projet plonge dans la spécification binaire FBX.

## Cahier des Charges

Consignes école :
- Parser le format binaire FBX (sans SDK officiel)
- Extraire les données mesh de base
- Gérer la décompression zlib
- Documenter le format

## Stack

- C++
- CMake
- FBX Binary (7.x)
- zlib (compression FBX)

## Notre Implémentation

Au-delà du parsing basique demandé, nous avons implémenté :

- **Parser FBX complet** : traversal bas-niveau de l'arbre de nodes, support v7.4 et v7.5
- **Extraction mesh avancée** : vertices, normals, UVs multiples, indices, bone weights pour skinning
- **Hiérarchie de scène complète** : reconstruction des relations parent-enfant avec transforms
- **Données d'animation** : extraction des curves et keyframes pour skeletal animation
- **Export dual format** : JSON lisible pour debug + binaire optimisé pour runtime

## Points Techniques

Structure binaire FBX :
- Arbre de nœuds
- Tableaux de propriétés avec types variés
- Compression zlib optionnelle
- Encodage custom pour strings et arrays

Extraction de données :
- Géométrie (vertices, polygones, normales)
- Mappings UV (canaux multiples)
- Poids de bones pour le skinning
- Courbes d'animation
