  ## Structure FBX Binaire

  Arbre de nœuds avec propriétés typées :

  ```
  FBX File
  └── Node Tree
      ├── Objects
      │   ├── Geometry (mesh data)
      │   ├── Deformer (bone weights)
      │   └── AnimationCurve (keyframes)
      └── Connections (scene hierarchy)
  ```

  ## Stack Détaillé

  | Tech | Version | Rôle |
  |------|---------|------|
  | C++ | 17 | Langage core |
  | CMake | 3.x | Build system |
  | zlib | 1.2 | Décompression FBX |
  | FBX Binary | 7.4/7.5 | Format cible |

  ## Parsing Implémenté

  - Traversal bas-niveau de l'arbre de nodes
  - Support v7.4 et v7.5
  - Décompression zlib optionnelle
  - Encodage custom strings et arrays

  ## Extraction de Données

  - **Géométrie** : vertices, polygones, normales, indices
  - **UVs** : canaux multiples
  - **Skinning** : bone weights pour skeletal animation
  - **Animation** : curves et keyframes
  - **Hiérarchie** : relations parent-enfant avec transforms

  ## Export

  - JSON lisible pour debug
  - Binaire optimisé pour runtime

# FBX Parser

Parser custom pour le format binaire FBX, permettant l'import de modèles 3D, animations squelettiques et hiérarchies de scène.

## Contexte

Projet école GTech3, 5 semaines. Équipe de 3, rôle : implémentation du parser binaire.

Comprendre les formats de fichiers 3D est essentiel en dev jeux. Ce projet plonge dans la spécification binaire FBX.

## Cahier des Charges École

- Parser le format binaire FBX (sans SDK officiel)
- Extraire les données mesh de base
- Gérer la décompression zlib
- Documenter le format

## Notre Implémentation

Au-delà du parsing basique demandé :

- **Parser FBX complet** : support v7.4 et v7.5
- **Extraction mesh avancée** : vertices, normals, UVs multiples, bone weights
- **Hiérarchie de scène** : reconstruction des relations parent-enfant
- **Données d'animation** : extraction curves et keyframes
- **Export dual format** : JSON debug + binaire runtime
