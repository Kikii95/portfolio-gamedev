# Template ECS

Moteur Entity Component System universel en C++20. Architecture modulaire, réutilisable pour tous types de jeux 2D.

## Contexte

Projet école GTech3 démarré en novembre 2025, développé en solo pour le cours de Jeu Libre.

Objectif : créer un moteur réutilisable comme base pour d'autres projets, avec une architecture propre et extensible.

## Stack

- C++20
- SFML 2.5+ (rendu)
- CMake 3.20+
- Patterns : ECS, Factory, Observer

## Architecture

Statistiques :
- 36 Components modulaires
- 35 Systems découplés
- 8+ Modules (core, transform, graphics, gameplay, inventory, interaction, AI, audio, network)

Types de Systems :
- Active : exécution continue (rendu, physique)
- Reactive : réponse aux événements
- TurnBased : jeux au tour par tour
- UI : interface utilisateur

Types de jeux supportés :
- Temps réel : FPS, platformers, action
- Tour par tour : stratégie, tactical RPG
- Grille : roguelikes, puzzles
- Hybride : combinaison des modes

## Fonctionnalités

- Architecture ECS pure : entités comme IDs, components comme données, systems pour le comportement
- Découplage total logique/données
- Memory pooling pour allocations fréquentes
- Spatial hashing pour collisions
- Thread-safe avec mutexes et atomics
- Memory-safe avec smart pointers et RAII
- Documentation complète

## Points Techniques

Optimisation collisions via spatial hashing :
- O(n) au lieu de O(n²)
- 125 entités : 25x plus rapide (125,000 → 1,000 checks)
- 1000 entités : 250x plus rapide (500,000 → 2,000 checks)
