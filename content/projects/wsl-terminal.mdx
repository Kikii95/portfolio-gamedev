# WSL Terminal

Émulateur de terminal moderne pour WSL2. Toutes les fonctionnalités d'un terminal pro, optimisé pour le développement sous WSL.

## Contexte

Projet perso démarré en décembre 2025. Développé en solo sur 2 semaines.

Windows Terminal est correct, mais je voulais quelque chose avec une meilleure intégration développeur : panneaux Git, gestion Docker, navigation projet, UI personnalisable — sans la lourdeur d'Electron.

## Stack

- Tauri 2.0
- Rust 1.77+ (backend)
- React 19, TypeScript 5, TailwindCSS 4 (frontend)
- xterm.js 5 avec addons
- Zustand (state management)

## Fonctionnalités

Terminal :
- Multi-onglets avec drag-and-drop
- Split panes horizontal/vertical redimensionnables
- Multi-fenêtres (détacher des onglets)
- Restauration de session automatique
- Recherche Ctrl+F

Outils dev :
- Panneau Git : branches, staging, commits, push/pull
- Panneau Docker : conteneurs et images
- Manager SSH : sauvegarde et connexion rapide
- Switcher de projets par catégorie
- Commandes rapides sauvegardées

UI :
- 6 thèmes (Catppuccin, Dracula, Nord, One Dark, Gruvbox, Tokyo Night)
- Palette de commandes Ctrl+Shift+P
- Mode Quake (terminal dropdown)
- Notifications fin de commandes longues

## Points Techniques

Léger comparé aux alternatives Electron : ~15MB d'installeur, 80-120MB de RAM. Le backend Tauri gère le spawn des shells et l'IPC, React gère l'UI.

Supporte toutes les distributions WSL installées (détection auto), PowerShell et CMD. Le switcher de projets s'intègre avec ma structure `~/projects/`.
