# Process Tracker

Application desktop pour gérer les services de développement local. Fini les processus dupliqués, conflits de ports et incertitudes sur ce qui tourne.

## Contexte

Projet perso démarré en décembre 2025. Développé en solo sur une semaine.

Problème résolu : quand on développe avec plusieurs services (backend, frontend, database), on se retrouve souvent avec des processus dupliqués, des conflits de ports, et aucune visibilité sur ce qui tourne.

## Stack

- Tauri 2.0
- Rust (backend)
- React, TypeScript (frontend)
- Cross-platform : Windows, Linux

## Fonctionnalités

- Start/Stop/Restart des services en un clic
- Dashboard de statut global
- Health checks HTTP
- Gestion des dépendances entre services
- Historique des exécutions
- Support WSL depuis Windows

## Points Techniques

Les services sont définis dans un fichier YAML simple. Le tracker surveille l'état des processus, gère l'arrêt propre et peut redémarrer automatiquement les services crashés.

Intégration WSL : peut exécuter des commandes dans les distributions WSL, permettant de gérer des services Linux depuis une GUI Windows.
