# Asset Pipeline Engine

Système de gestion d'assets robuste pour moteurs de jeu : chargement, cache, hot-reload et résolution de dépendances.

## Contexte

Projet école GTech3, 5 semaines. Équipe de 3, rôle : architecte du pipeline core.

Les jeux modernes nécessitent une gestion d'assets sophistiquée. Ce projet implémente un pipeline production-ready pour les ressources de jeu.

## Stack

- C++
- CMake
- Sérialisation binaire custom
- APIs système (file watching)

## Fonctionnalités

- Chargement async avec callbacks
- Reference counting : gestion automatique du cycle de vie des ressources
- Hot reload : mise à jour live des assets en développement
- Graphe de dépendances : rechargement automatique des assets dépendants
- Compression LZ4 pour les assets packés

## Points Techniques

Resource Manager centralisé gérant :
- Enregistrement et lookup d'assets
- Budget mémoire
- Prioritisation du chargement
- Garbage collection

Pipeline de build offline pour :
- Compression textures (formats BCn)
- Optimisation mesh
- Compilation shaders
- Bundling d'assets
