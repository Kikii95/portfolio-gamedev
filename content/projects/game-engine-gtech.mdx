# Custom Game Engine

Moteur de jeu développé from scratch avec renderer OpenGL moderne et architecture entity-component-system.

## Contexte

Projet école GTech3, développé tout au long de l'année (6 mois). Projet individuel.

Construction progressive des systèmes fondamentaux d'un moteur de jeu pour comprendre l'architecture des moteurs temps réel et le graphisme 3D.

## Stack

- C++20
- OpenGL 4.5, GLSL
- Dear ImGui (éditeur)
- GLM (maths)
- CMake

## Fonctionnalités

- Renderer OpenGL : pipeline de rendu différé
- Architecture ECS : gestion d'entités data-oriented
- Scene Graph : système de transformations hiérarchiques
- Éditeur ImGui : level editor intégré
- Pipeline d'assets : chargement modèles et textures
- Système de shaders : hot-reload GLSL

## Points Techniques

Pipeline de rendu différé avec G-buffer pour la géométrie, passe d'éclairage multi-lumières, effets post-process (bloom, tone mapping), shadow mapping.

ECS data-oriented : entités = IDs, components = données pures, systems = comportements. Séparation claire pour la performance et la maintenabilité.
