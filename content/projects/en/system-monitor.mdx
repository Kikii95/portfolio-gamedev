  ## Architecture

  Layered design separating collection, history and rendering:

  ```
  /proc/* + sysinfo + nvml
           │
           ▼
  ┌─────────────────┐
  │   Collectors    │  ← CPU, Memory, GPU, Network, Disk, Process
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │ MetricsHistory  │  ← RingBuffer<60> for sparklines
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │    Widgets      │  ← ratatui::Widget for each metric
  └────────┬────────┘
           │
           ▼
  ┌─────────────────┐
  │ ratatui::Frame  │  ← Diff rendering, terminal output
  └─────────────────┘
  ```

  ## Performance Metrics

  - **CPU**: 0.0% utilization at idle
  - **RAM**: ~4MB footprint
  - **Binary**: 1.7MB (stripped, LTO enabled)
  - **Startup**: <100ms

  ## Optimizations

  - **Lazy Updates**: refresh only if `tick_elapsed`
  - **Diff Rendering**: ratatui compares buffers, redraws only changed cells
  - **Batch /proc Reads**: single `refresh_all()` for all metrics
  - **SmartString**: stack-allocated for small strings (process names)

# System Monitor

Terminal system performance monitor. Inspired by htop/btop/nvtop but much lighter.

## Context

Personal project started in November 2025. Solo development over one week.

Goal: a minimalist system monitor, keyboard-driven, that consumes almost nothing while remaining visually pleasing.

## What it does

**Performance**:
- Near-zero consumption: 0.0% CPU, ~4MB RAM
- Single 1.7MB binary
- Instant startup (<100ms)

**Monitoring**:
- CPU, RAM, Swap in real-time
- NVIDIA GPU: usage, VRAM, temperature, power draw
- 60-second sparklines for all metrics
- Top 5 processes by consumption

**Interface**:
- 9 visual themes (Matrix, Dracula, Nord, Tokyo Night, Gruvbox, etc.)
- Full keyboard navigation
- Persistent config saved in one keypress
